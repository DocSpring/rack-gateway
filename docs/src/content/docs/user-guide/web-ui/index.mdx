The Rack Gateway web UI is the primary way to inspect rack state, manage users, review audit logs, and configure gateway settings.

## Accessing the Web UI

### From the CLI

```bash
rack-gateway web
```

### Direct URL

```
https://gateway.example.com
```

You'll be redirected to Google OAuth if you aren't signed in.

## Navigation Overview

The sidebar contains these sections (some are admin-only):

| Section | Description | Access |
|---------|-------------|--------|
| **Rack** | Rack metadata and status | All users |
| **Apps / Processes / Instances / Builds / Releases** | Convox operational views | Role-based |
| **Users** | User list and details | All users (admin actions only) |
| **API Tokens** | Token list and management | All users (create/edit requires Admin or Deployer) |
| **Deploy Approvals** | Pending approvals | Admin only |
| **Audit Logs** | Audit trail and export | All users (read-only) |
| **Account Security** | MFA + trusted devices | All users |
| **Integrations** | Slack, GitHub, CircleCI | Admin only |
| **Settings** | Global security settings | Admin only |
| **Background Jobs** | Job queue visibility | Admin only |
| **Configure CLI** | Quick CLI setup dialog | All users |

<Aside type="note">
Non-admins can browse most read-only views, but only admins can change users, integrations, or global settings.
</Aside>

## App Settings

App-level settings live under **Apps → `{app}` → Settings**. This is where admins configure:

- VCS/CI integration per app (GitHub + CircleCI)
- Protected and secret environment variables
- Approved deploy commands
- Service image patterns used for deploy verification

## Signing Out

Use the **Logout** button at the bottom of the sidebar.
