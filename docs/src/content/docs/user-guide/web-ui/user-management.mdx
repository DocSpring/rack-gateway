Administrators can manage user accounts, assign roles, and control access through the Users section of the web interface.

<Aside type="note">
User management is only available to administrators.
</Aside>

## Viewing Users

Navigate to **Users** in the sidebar to see all users:

| Column | Description |
|--------|-------------|
| **Email** | User's email address |
| **Role** | Current assigned role |
| **MFA** | Whether MFA is enabled |
| **Status** | Active or locked |
| **Created** | When the user was added |
| **Last Login** | Most recent authentication |

## Adding Users

<Steps>

1. Click **Add User** in the top-right

2. Enter the user's email address
   - Must match your allowed domain (e.g., `@example.com`)

3. Select a role:
   - **Admin** - Full access
   - **Deployer** - Deploy applications
   - **Ops** - Operate applications
   - **Viewer** - Read-only access

4. Click **Create User**

</Steps>

The user will receive an email notification with instructions to log in.

## Changing User Roles

<Steps>

1. Find the user in the list

2. Click the **Edit** button (or the role dropdown)

3. Select the new role

4. Confirm the change

</Steps>

<Aside type="caution">
Role changes take effect immediately. The user's current session will use the new permissions.
</Aside>

## Locking Users

To temporarily disable a user's access:

<Steps>

1. Find the user in the list

2. Click **Lock Account**

3. Confirm the action

</Steps>

Locked users:
- Cannot log in
- Existing sessions are invalidated
- Account remains in the system

To unlock, click **Unlock Account**.

## Deleting Users

<Steps>

1. Find the user in the list

2. Click **Delete**

3. Confirm the deletion

</Steps>

<Aside type="danger">
Deletion is permanent. The user's audit history is preserved, but they will need to be re-added to regain access.
</Aside>

## Resetting User MFA

If a user loses access to their MFA device:

<Steps>

1. Find the user in the list

2. Click **Reset MFA**

3. Confirm the action

</Steps>

This removes all MFA methods from the user's account. They can then enroll new methods on their next login.

## Bulk Operations

For multiple users:

1. Select users using the checkboxes
2. Choose an action from the **Bulk Actions** menu:
   - Change role
   - Lock accounts
   - Unlock accounts

## Filtering and Search

Use the search box to filter users by:
- Email address
- Role

## User Details

Click a user's email to view their details:

- Account information
- MFA methods enrolled
- Recent login history
- Active sessions
