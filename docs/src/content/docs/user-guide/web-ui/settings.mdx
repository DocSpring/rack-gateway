The Settings section allows administrators to configure gateway behavior, security policies, and integrations.

<Aside type="note">
Settings management is only available to administrators.
</Aside>

## Accessing Settings

Navigate to **Settings** in the sidebar to view and modify configuration.

## Security Settings

### Session Timeout

Control how long user sessions remain active:

| Setting | Description |
|---------|-------------|
| **Timeout (minutes)** | Session expires after this many minutes of inactivity |
| **Recommended** | 5-30 minutes for security-sensitive environments |

<Steps>

1. Enter the desired timeout in minutes

2. Click **Save**

3. New timeout applies to new sessions

</Steps>

### MFA Requirements

Configure when MFA is required:

| Setting | Description |
|---------|-------------|
| **Require MFA for sensitive actions** | Enforce MFA for destructive operations |
| **Required actions** | Delete apps, promote releases, manage users |

When enabled, users without MFA enrolled will be prompted to set it up before performing sensitive operations.

### Destructive Actions Guard

Control whether destructive operations are allowed:

| Setting | Description |
|---------|-------------|
| **Allow destructive actions** | Enable/disable app deletion, process termination |
| **Default** | Disabled (safer) |

<Aside type="caution">
When disabled, even administrators cannot delete applications or terminate processes. Enable temporarily when needed.
</Aside>

## Environment Variables

### Protected Variables

Configure which environment variables are masked in responses:

```
DATABASE_URL
RACK_TOKEN
APP_SECRET_KEY
AWS_SECRET_ACCESS_KEY
```

Protected variables:
- Are masked in API responses
- Cannot be modified via the gateway
- Are redacted in audit logs

<Steps>

1. Click **Edit Protected Variables**

2. Add variable names (one per line)

3. Click **Save**

</Steps>

### Viewing Environment

View current gateway environment (with sensitive values masked):

| Variable | Value |
|----------|-------|
| DOMAIN | gateway.example.com |
| DATABASE_URL | [PROTECTED] |
| RACK_ALIAS | production |

## Notifications

### Email Notifications

Configure email settings:

| Setting | Description |
|---------|-------------|
| **Enabled** | Send email notifications |
| **From address** | Sender email |
| **Admin BCC** | CC administrators on notifications |

Events that trigger emails:
- New user added
- API token created
- Deploy approval requested

### Slack Integration

Connect to Slack for notifications:

<Steps>

1. Click **Connect Slack**

2. Authorize the Slack app

3. Select notification channel

4. Choose events to notify

</Steps>

## Deploy Approvals

Configure the deploy approval workflow:

| Setting | Description |
|---------|-------------|
| **Enabled** | Require approval for deploys |
| **Approval timeout** | How long approvals remain valid |
| **Required approvers** | Minimum approvers needed |

See [Deploy Approvals](/integrations/deploy-approvals/) for detailed configuration.

## Sentry Integration

Configure error tracking:

| Setting | Description |
|---------|-------------|
| **Backend DSN** | Sentry project DSN |
| **Frontend DSN** | Browser error tracking |
| **Environment** | Environment tag |
| **Traces sample rate** | Performance monitoring (0-1) |

### Testing Sentry

When enabled, test buttons appear:

- **Test Backend Error** - Triggers a test exception
- **Test Frontend Error** - Triggers a browser error
- **Test Performance** - Sends a test transaction

## Audit Settings

### Retention

Configure log retention:

| Setting | Description |
|---------|-------------|
| **Retention days** | How long to keep logs |
| **Recommended** | 400 days (SOC 2) |

### WORM Anchoring

Configure tamper-evident storage:

| Setting | Description |
|---------|-------------|
| **Enabled** | Write anchors to S3 |
| **Bucket** | S3 bucket name |
| **Interval** | How often to write anchors |
| **Retention** | Object Lock retention period |

## Applying Changes

Most settings take effect immediately. Some require:

- **Session timeout**: Affects new sessions only
- **MFA requirements**: May prompt users on next sensitive action
- **Protected variables**: Affects new API responses

## Settings History

View recent settings changes:

1. Go to **Audit Logs**
2. Filter by `action:settings.*`
3. See who changed what and when

## Exporting Settings

Export current configuration:

<Steps>

1. Click **Export Settings**

2. Download JSON file

3. Use for documentation or disaster recovery

</Steps>
