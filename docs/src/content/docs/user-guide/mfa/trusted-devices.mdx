Trusted devices let the gateway remember a browser after a successful MFA verification.

## How It Works

- A trusted-device cookie is set after verification
- Future logins skip MFA until the TTL expires (default 30 days)
- Trust is browser-specific

## Trusting a Device

You can trust a device in two ways:

### During MFA Verification

<Steps>

1. Complete MFA verification
2. Check **Trust this device for 30 days**
3. Finish sign-in

</Steps>

### From Account Security

<Steps>

1. Go to **Account Security**
2. In **Trusted Devices**, click **Trust This Device**

</Steps>

## Managing Trusted Devices

The **Trusted Devices** table shows:

- Device label
- Last used time
- IP address
- Expiration

To revoke a device, click **Revoke** next to the entry.

## Configuration

Admins can set the trusted-device TTL in **Settings → MFA Requirements** (`mfa_trusted_device_ttl_days`).

<Aside type="note">
Setting the TTL to a low value increases security but reduces convenience.
</Aside>
